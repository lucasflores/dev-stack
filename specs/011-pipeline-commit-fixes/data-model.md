# Data Model: Pipeline Commit Fixes

**Feature**: `011-pipeline-commit-fixes`
**Date**: 2026-03-12

## Entities

### ParsedCommitMessage

Extracted clean commit message after processing an `AgentResponse`.

| Field | Type | Description |
|-------|------|-------------|
| `subject` | `str` | First line of the commit message (≤ 72 chars recommended) |
| `body` | `str \| None` | Optional body text after the blank line separator |
| `raw_content` | `str` | Full extracted text before subject/body split |
| `extraction_method` | `ExtractionMethod` | How the message was extracted (`CODE_FENCE`, `PLAIN_TEXT`) |
| `source_hash` | `str` | SHA-256 of the raw agent response (for debug logging) |

**Validation rules**:
- `raw_content` MUST NOT be empty or whitespace-only
- `subject` MUST NOT exceed 200 characters (likely not a valid subject line)
- `raw_content` MUST NOT contain obvious tool-use artifacts at line starts

**State transitions**: N/A — immutable value object.

### ExtractionMethod (Enum)

| Value | Description |
|-------|-------------|
| `CODE_FENCE` | Message extracted from the last code-fenced block in agent response |
| `PLAIN_TEXT` | No code fences found; full response used as-is |

### HookContext *(design-time concept)*

Information about which git hook is executing and the arguments git passed. This is a **design-time entity** for documentation and reasoning — it is NOT instantiated as a Python class at runtime. The runtime mechanism uses environment variables (`DEV_STACK_HOOK_CONTEXT`, `DEV_STACK_MESSAGE_FILE`) and function parameters (`source`, `commit_sha`) passed directly to `run_prepare_commit_msg_hook()`.

| Field | Type | Runtime mechanism |
|-------|------|-------------------|
| `hook_name` | `str` | Env var `DEV_STACK_HOOK_CONTEXT` |
| `message_file` | `Path \| None` | Env var `DEV_STACK_MESSAGE_FILE` / function param |
| `source` | `str \| None` | Function param from git's `$2` |
| `commit_sha` | `str \| None` | Function param from git's `$3` |

**Decision matrix for commit-message generation**:

| `source` | Generate? | Reason |
|----------|-----------|--------|
| `None` (absent) | YES | Plain `git commit` — no message provided |
| `template` | YES | Template present but user hasn't written content yet |
| `message` | NO | User provided `-m "msg"` or `-F file` |
| `commit` | NO | Amending/reusing existing message |
| `merge` | NO | Merge message auto-generated by git |
| `squash` | NO | Squash message auto-generated by git |

### StagedSnapshot

A lightweight hash of the staging area state for integrity verification.

| Field | Type | Description |
|-------|------|-------------|
| `diff_hash` | `str` | SHA-256 of `git diff --cached` output |
| `file_list_hash` | `str` | SHA-256 of `git diff --cached --name-only` sorted output |
| `captured_at` | `float` | `time.perf_counter()` timestamp |

**Usage**: Captured before and after each agent-powered stage. If `diff_hash` differs, the stage violated staged content protection.

### AgentInvocationMode (Enum)

Controls agent permission level based on execution context.

| Value | Description |
|-------|-------------|
| `SANDBOXED` | No filesystem write access; used during hook-context pipeline execution |
| `FULL_ACCESS` | Current behavior with `--allow-all`; used for CLI `dev-stack run` invocations |

**Decision logic**: `SANDBOXED` when `StageContext.hook_context` is set; `FULL_ACCESS` otherwise.

### AdvisoryDocSuggestion

Captures file-change suggestions from agent stages without applying them.

| Field | Type | Description |
|-------|------|-------------|
| `stage_name` | `str` | Which stage produced the suggestion (e.g., `docs-narrative`) |
| `content` | `str` | The raw agent output containing suggested changes |
| `target_path` | `str` | Relative path of the file the agent wanted to modify |
| `timestamp` | `str` | ISO 8601 timestamp |

**Storage**: Appended to `.dev-stack/pending-docs.md` as markdown sections.

## Relationships

```
AgentResponse (raw stdout)
    │
    ▼  response_parser.extract_commit_message()
ParsedCommitMessage
    │
    ▼  upsert_trailers()
str (final commit text with trailers)
    │
    ▼  write to message_file ($1)
Git reads and records the commit

HookContext
    │
    ├─▶ Determines stage selection (pre-commit vs prepare-commit-msg)
    ├─▶ Determines AgentInvocationMode (SANDBOXED vs FULL_ACCESS)
    └─▶ Determines whether to generate message (source check)

StagedSnapshot (before stage)
    │
    ▼  stage execution
StagedSnapshot (after stage)
    │
    ▼  compare diff_hash
    ├─ Match: continue
    └─ Mismatch: ERROR — stage violated integrity
```
