# Data Model: APM Module Swap

**Feature**: 013-apm-module-swap | **Date**: 2026-03-24

## Entities

### 1. APMModule

**Type**: Class (extends `ModuleBase`)
**Location**: `src/dev_stack/modules/apm.py`

| Field | Type | Description |
|-------|------|-------------|
| `NAME` | `str` (class const) | `"apm"` |
| `VERSION` | `str` (class const) | `"0.1.0"` |
| `DEPENDS_ON` | `Sequence[str]` (class const) | `()` — no module dependencies |
| `MANAGED_FILES` | `Sequence[str]` (class const) | `("apm.yml", "apm.lock.yaml")` |
| `MIN_APM_VERSION` | `str` (class const) | `"0.8.0"` |
| `DEFAULT_SERVERS` | `tuple[str, ...]` (class const) | 5 default MCP server registry URIs |

**Methods**:

| Method | Signature | Description |
|--------|-----------|-------------|
| `install()` | `(force: bool = False) -> ModuleResult` | Bootstrap `apm.yml` (with conflict prompt), run `apm install` |
| `uninstall()` | `() -> ModuleResult` | Remove `apm.yml` and `apm.lock.yaml` |
| `update()` | `() -> ModuleResult` | Re-run `apm install` (respects existing `apm.yml`) |
| `verify()` | `() -> ModuleStatus` | Check: APM on PATH, version OK, `apm.yml` exists, lockfile fresh |
| `preview_files()` | `() -> dict[Path, str]` | Return default `apm.yml` template content |
| `audit()` | `(fmt: str = "text", output: Path \| None = None) -> ModuleResult` | Run `apm audit` with format/output options |
| `_check_apm_cli()` | `() -> tuple[bool, str]` | Verify APM binary exists and meets minimum version |
| `_bootstrap_manifest()` | `(strategy: str) -> Path` | Create/merge/skip `apm.yml` based on user choice |
| `_run_apm()` | `(args: list[str]) -> subprocess.CompletedProcess` | Execute APM CLI subprocess |
| `_parse_install_result()` | `(result: subprocess.CompletedProcess) -> ModuleResult` | Parse stdout/stderr for per-server status |

**Relationships**:
- Registered in `_MODULE_REGISTRY` via `@register_module` decorator
- Listed in `DEFAULT_GREENFIELD_MODULES` (replaces `mcp_servers`)
- Instantiated by `instantiate_modules()` during init pipeline

### 2. APM Manifest (`apm.yml`)

**Type**: YAML file (project root)
**Format**: APM-native format

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `str` | yes | Project identifier |
| `version` | `str` | yes | Semantic version |
| `dependencies.mcp` | `list[str \| dict]` | yes | MCP server references (registry URIs or self-defined) |
| `target` | `str` | no | Compilation target (`copilot`, `claude`, `all`) |

**Validation rules**:
- `name` must be non-empty
- `dependencies.mcp` must contain at least one entry
- Each MCP entry is either a string (registry URI) or a dict with `name` key

**State transitions**:
- **Not exists** → `_bootstrap_manifest("overwrite")` → **Created with defaults**
- **Exists** → User prompt → **Skip** (no change) / **Merge** (add missing defaults) / **Overwrite** (replace)
- **Created/Merged** → `apm install` → **Locked** (lockfile generated)

### 3. APM Lockfile (`apm.lock.yaml`)

**Type**: YAML file (project root, generated by APM)
**Ownership**: APM CLI (dev-stack read-only)

| Field | Type | Description |
|-------|------|-------------|
| `lockfile_version` | `str` | Lock format version |
| `generated_at` | `str` (ISO 8601) | Generation timestamp |
| `apm_version` | `str` | APM CLI version that generated it |
| `dependencies` | `dict[str, dict]` | Resolved packages with commit SHAs, deployed files |
| `mcp_servers` | `list[str]` | List of installed MCP server names |

**Dev-stack interactions**:
- `verify()`: Check file exists
- `verify()`: Compare mtime against `apm.yml` to detect staleness
